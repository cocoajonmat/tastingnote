# 학습 로드맵

## 목표

단순히 기능을 완성하는 것이 아니라, **왜 이렇게 만드는지 스스로 설명할 수 있는 개발자**가 되는 것.

AI가 코드를 짜주는 시대에 필요한 역량은 "코드를 빠르게 타이핑하는 것"이 아니라:
- 문제를 정확히 정의하는 것
- 트레이드오프를 인식하고 결정하는 것
- 코드가 왜 그렇게 됐는지 설명하는 것
- 실패했을 때 어디가 문제인지 찾는 것

---

## 진행 방식

1. **유저가 설계 먼저** — 기능 시작 전 "엔티티는 이렇게, API는 이렇게, 흐름은 이렇게" 먼저 말하기
2. **Claude는 검토자** — 문제점, 누락, 트레이드오프를 질문으로 이끌어냄. 답을 바로 주지 않음
3. **설명할 수 있어야 아는 것** — 구현 후 본인 말로 설명 못 하면 다시 짚고 넘어감
4. **코드 직접 수정** — 개념을 이해했으면 직접 코드로 옮겨봄

---

## AI 시대에 필요한 역량 vs 불필요한 역량

### 필요한 것
- **문제 정의 능력** — "무엇을 만들어야 하는가"를 정확히 말하는 것. AI는 이걸 대신 못 해줌
- **설계 사고** — 엔티티 관계, API 흐름, 예외 케이스를 머릿속에서 그리는 것
- **트레이드오프 판단** — A안 vs B안 중 이 상황에서 무엇이 더 적합한지 결정하는 것
- **디버깅 능력** — 에러가 나면 어디서 왜 나는지 추적하는 것
- **코드 읽는 능력** — AI가 짜준 코드를 이해하고 검토하는 것

### 덜 중요해진 것
- 문법 암기 — AI가 다 써줌
- 보일러플레이트 작성 — AI가 다 써줌
- 특정 프레임워크 API 외우기 — 검색하면 됨

## 현재 코드베이스 학습 현황

### 완료
- `@Transactional` — 클래스/메서드 레벨, readOnly, dirty checking
- N+1 문제 — LAZY vs JOIN FETCH, IN 쿼리 배치 조회
- TOCTOU 레이스 컨디션 — existsBy + save 패턴, DataIntegrityViolationException 방어
- `signUp` 코드 직접 수정 완료
- OAuth 유저 로컬 로그인 시도 — BCrypt null 처리, BusinessException 커스텀 메시지, Provider.getDescription()
- S3 + DB 비원자성 — DB 먼저 삭제 후 S3, 아웃박스 패턴 개념, 트레이드오프 이해
- Stateless JWT 한계 — Redis 블랙리스트 원리, TTL, 인메모리 vs 디스크 속도 차이
- S3 키 URL 역추출 취약점 — s3Key DB 별도 저장으로 해결, NoteImage/User 엔티티 수정 완료

### 진행 중 (다음 세션부터)
- 없음 — 코드베이스 학습 주제 전체 완료

### 앞으로 기능 구현 시 직접 설계해볼 주제
- **Like 기능** — 중복 좋아요 방어 (TOCTOU), likeCount 동시성 문제, deleteNote 연동
- **Tag 기능** — 태그 사용 횟수 카운팅 방식 (컬럼 vs 실시간 COUNT), 자동완성 쿼리
- **유저 탈퇴** — Soft Delete 흐름, 닉네임 unique 처리, Access Token 잔존 문제
- **술 상세 페이지** — 집계 쿼리 설계, 캐싱 필요 여부 판단

## 기능 구현 순서 (context.md 기준)

| 순서 | 기능 | 핵심 학습 주제 |
|------|------|---------------|
| 1 | Like | 동시성, likeCount 비정규화, FK 삭제 순서 |
| 2 | Tag | 카운팅 전략, 자동완성 쿼리 |
| 3 | 유저 탈퇴 | Soft Delete, 토큰 만료 전략 |
| 4 | Flyway | ddl-auto validate, 마이그레이션 관리 |
| 5 | 술 상세 페이지 | 집계 쿼리, 성능 고려 |
